## API
В этом модуле собраны функции, обращающиеся к back-end REST API.
Они должны возвращать сырые данные и выкидывать исключения, если что-то пошло не так.
Здесь НЕТ обработки данных! Максимум, обработка исключений. Вся логика обработки должна быть прописана в services.

В `ApiConfig.ts` прописываются все URL.

Формат файлов - `ts`.

Файлы с суффиксом `.mock.ts` - демо-файлы. Они нужны для отладки или для тех случаев, когда бэкенд не успевает за реализацией фронтенда.
Такой файл должен содержать точно те же функции, что и соответствующий ему файл без суффикса. Все функции в нём не посылают реальные запросы.
Они возвращают "демо-данные", сохраняя нужный формат. Для теста в необходимом `import` к имени файла дописывается `.mock`.

Такие файлы необязательны, но желательны.
